Coicons - A lightweight, CSS icon library. Similar to Emoji but with a touch of elegance

Coicons is a lightweight and flexible CSS powered library that gives an elegent and colourful touch to your website. Baked to enhance design and experience - the library is fast loading and robust. Implementing both png and svg formats, coicons is tweekable and easy to use. Take a bite and enjoy an unforgetable popping taste of color ;)

Installation

Coicons CSS can be used for both development and production environments. For development, first download library files into the local development environment. This gives flexibility in testing and tweeking .

Developement
1. Download the library from its Github repository and include the files in the local development environment.
2. Add the coicons.css file or the minified version inside the HTML document's <head> tags.
Production
1. Simply click to copy the resource link to clipboard: <link href="https://goo.gl/9CPwDt" rel="stylesheet"> then add inside the HTML document's <head> tags.
Usage

Coicons implement the <i> tag. Therefore to include coicons in the HTML first open up the <i> tag with a default class .coi. Within the class parameter, after the default .coi class, add another class .coi-{name-of-coicon}. This last class calls for the coicon specified.

By default, the coicons will be served as PNGs. To use SVG instead, the .coi-svg class is used instead of the default coi. This will scale better, but has larger file size.

Coicons come in sizes which can be implemented using the (default) .coi, (small) .coi-sm, and (large).coi-lg classes. For SVGs, .coi-svg, .coi-svg-sm, .coi-svg-lg classes are used.

Example (PNGs & SVGs)
<i class="coi coi-32bit"></i>
<i class="coi coi-svg coi-32bit"></i>
Example (Sizes: default, small & large)
<i class="coi coi-32bit"></i>
<i class="coi coi-sm coi-32bit"></i>
<i class="coi coi-svg-lg coi-32bit"></i>
Tweeks

Creating custom CSS rules can manage tweeking of coicons' default sizes. Coicons are set to the below sizes by default and tweeking on custom CSS rules can overide them.

Default : .coi and .coi-svg
{
width: 4em;
height: 4em;
margin-top: 1px;
...}
Small: .coi-sm and .coi-svg-sm
{
width: 3.5em;
height: 3.5em;
margin-top: 1px;
...}
Large : .coi-lg and .coi-svg-lg
{
width: 5em;
height: 5em;
margin-top: 1px;
...}
